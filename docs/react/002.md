# React tanfolyam - 2. alkalom

## Rövid célkitűzés

Ebben a rövid bevezetőben a React alap gondolkodásmódját és gyakorlati használatát mutatjuk be. A cél: érthetően bemutatni, miért használunk React-et, hogyan épülnek fel a komponensek, és hogyan kapcsolódunk adatforrásokhoz (API-khoz).

A Korábbi `Webes alapok` anyagokra építünk: ha még nem vagy biztos a HTML/CSS/JS alapokban, érdemes előbb azt átismételni: _**[Webes alapok](/docs/webes-alapok/intro.md)**_.

## Mi az a React?

A React egy komponens-alapú JavaScript könyvtár, amely deklaratív módon segít felhasználói felületek létrehozásában. A fő gondolatok röviden:

- UI = f(state): a felület az alkalmazás állapotának egy leképezése.
- Deklaratív: nem "mit módosíts", hanem "hogyan nézzen ki" adott állapotban.
- Komponensek: a UI kis újrafelhasználható egységekből áll.
- Egyirányú adatfolyam: adatok a szülőtől a gyerek(ek) felé áramlanak.

A Kir-Dev projektjeiben React mellett gyakran használunk Next.js-t (routing, szerveroldali renderelés, optimalizálás) és Tailwind CSS-t a stílushoz.

## Gyors példa — komponens (JSX és props)

Egy egyszerű, bemutató komponens, ami megmutatja a props használatát és a JSX szerkezetét:

```jsx
// src/components/HelloCard.jsx
import React from 'react';

function HelloCard({ name }) {
  return (
    <div className="p-4 border rounded shadow-sm">
      <h2 className="text-xl font-semibold">Sziasztok, {name}!</h2>
      <p>Ez egy egyszerű React komponens props-szal.</p>
    </div>
  );
}

export default HelloCard;
```

Megjegyzés: a fenti példa egy kis Tailwind-osztályokat használ; ha nem használjuk a Tailwindet, hagyjuk el a className attribútumot vagy írjunk saját CSS-t.

## Adatbetöltés példa (hooks: useState + useEffect)

Gyakori feladat: adat betöltése egy API-ról és listázása. Itt egy rövid példa, ami szemlélteti a hook-ok használatát:

```jsx
// src/components/DataFetcher.jsx
import React, { useEffect, useState } from 'react';

function DataFetcher() {
  const [items, setItems] = useState([]);
  const [loading, setLoading] = useState(true);
  const [error, setError] = useState(null);

  useEffect(() => {
    let cancelled = false;

    async function load() {
      try {
        const res = await fetch('/api/items'); // példa endpoint
        if (!res.ok) throw new Error('Hálózati hiba');
        const data = await res.json();
        if (!cancelled) setItems(data);
      } catch (e) {
        if (!cancelled) setError(e.message);
      } finally {
        if (!cancelled) setLoading(false);
      }
    }

    load();

    return () => {
      cancelled = true; // cleanup: elkerüljük az állapotfrissítést unmount után
    };
  }, []);

  if (loading) return <div>Betöltés...</div>;
  if (error) return <div className="text-red-600">Hiba: {error}</div>;

  return (
    <ul>
      {items.map((it) => (
        <li key={it.id}>{it.title}</li>
      ))}
    </ul>
  );
}

export default DataFetcher;
```

Tippek:
- A fetch példában szereplő `/api/items` helyett használhatod a Node vagy Spring backend végpontját is.
- Kezeld a hibákat és a betöltési állapotot; soha ne hagyd figyelmen kívül az async műveletek cleanup-ját.

## React gondolkodásmód — összefoglaló

- Gondolkodj komponensekben: bontsd le a felületet kis, jól definiált részekre.
- Tartsd egyszerűen az állapotot: csak azt tartsd local state-ben, ami tényleg lokális.
- Gondoskodj az oldalsó hatásokról (side effects) — használj hook-okat, és tisztítsd őket.
- Prop-drilling helyett fontold meg a contextet vagy egy állapotkezelő könyvtárat (kis projektnél gyakran elég a Context + hooks, nagyobb projektnél Redux/ Zustand/ Recoil stb.).

## Styling a projektben

A Kir-Dev projektekben Tailwind CSS a preferált megoldás (utility-first). Ennek előnyei:

- Gyors prototípus-készítés
- Kis konzisztens osztálykészlet
- Könnyű komponensekhez igazítani

Ha nem Tailwind-et használsz, írhatsz modul CSS-t vagy styled-components-et is; a lényeg a konzisztencia a csapaton belül.

## Hogyan kapcsolódunk a backendhez?

A frontend tipikus feladatai backend integráció esetén:

- REST vagy GraphQL végpontok hívása (fetch/axios/apollo)
- JSON feldolgozása és lokalizált állapotba helyezése
- Autentikáció kezelése (tokenek, cookie-k, session)
- Hibakezelés és újrapróbálkozás (retry), szükség esetén cache-elés

A következő alkalomtól gyakorlati kódot írunk Node backendhez, illetve a Spring ticketing API-hoz.

## Tanulási célok a következő alkalomra

- Next.js projekt indítása (alap routing, oldal és komponens struktúra)
- Alap komponensek implementálása (listák, űrlapok, gombok)
- Backend csatlakoztatása: lista lekérése és megjelenítése
- Egyszerű állapotkezelés és formok kezelése
